# Руководство по разработке и поддержке фронтенда (Web)

Этот документ содержит подробную информацию о веб-интерфейсе `omsu_setka`, его архитектуре и процессах разработки.

## Технологический стек

- **Фреймворк:** React 19 (TypeScript)
- **Сборщик:** Vite
- **Управление состоянием:** Zustand (избранное, история поиска)
- **Работа с данными:** TanStack Query (React Query)
- **Стилизация:** Vanilla CSS Modules (дизайн Glassmorphism)
- **Иконки:** Lucide-React
- **PWA:** Плагин Vite PWA

## Основная архитектура

Фронтенд спроектирован по принципу mobile-first с акцентом на высокую скорость взаимодействия и отзывчивость.

### Ключевые компоненты

1. **Состояние (`Zustand`):**
   - Хранит избранные объекты (группы, преподаватели, аудитории).
   - Управляет единой историей поиска (ограничена 5 последними запросами).
2. **Запросы данных (`React Query`):**
   - Эффективно кэширует расписание и данные о здоровье системы.
   - Обеспечивает фоновое обновление данных при изменении статуса.
3. **Стилизация (`CSS Modules`):**
   - Использует переменные CSS для управления темами (светлая/темная).
   - Реализует эффект Glassmorphism с использованием `backdrop-filter` и мягких свечений.

### Навигация и страницы

- **Главная:** Поиск с автодополнением, избранное и индикатор статуса.
- **ScheduleView:** Интерактивное расписание с переключением дней, недель и фильтрацией подгрупп.
- **TutorsPage:** Специализированный поиск и просмотр данных о преподавателях.
- **StatusPage:** Панель мониторинга и история инцидентов оригинального сервера.

## Особенности интерфейса (UX/UI)

- **Оптимизация для десктопа:** Ширина контента ограничена 1100px. Поля поиска центрированы и имеют максимальную ширину 600px.
- **Цветовая кодировка:** Типы занятий (Практика — красный, Лабораторная — зеленый, Лекция — индиго) выделены цветом только в текстовых метках.
- **Заглушки для окон:** Расписание автоматически вставляет карточки «Нет занятия» в пустые слоты до конца учебного дня.
- **Анимации:** Используется `framer-motion` для плавных переходов между состояниями.

## Обслуживание и разработка

### Темы
Для изменения цветовой гаммы, отступов или скоростей анимации редактируйте `src/styles/variables.css`.

### Настройка PWA
Манифест и параметры офлайн-режима настраиваются в `site.webmanifest` и конфигурации плагина PWA.

### Сборка и развертывание
1. Установка: `npm install`
2. Запуск для разработки: `npm run dev`
3. Сборка для продакшена: `npm run build`

### Обработка ошибок
Используется глобальный `ErrorBoundary` для перехвата и отображения дружелюбного интерфейса при сбоях в компонентах.
