# omsu_setka Core (Backend)

Серверная часть проекта `omsu_setka` — это высокопроизводительный BFF (Backend-for-Frontend), написанный на Go. Он отвечает за агрегацию, кэширование и быструю отдачу данных расписания ОмГУ.

## Особенности бэкенда

- **Многоуровневое кэширование**:
  - **L1 (Memory)**: sync.Map с готовыми JSON-блоками (отдача < 1мс).
  - **L2 (Persistent)**: SQLite для хранения данных между перезапусками.
- **Фоновая синхронизация**: Автоматическое обновление «горячих» расписаний без участия пользователя.
- **Мгновенный поиск**: Использование префиксного дерева (Trie) в памяти для автодополнения.
- **Zero-Allocation**: Минимизация аллокаций памяти при обработке запросов для стабильной работы под нагрузкой.

## Технологический стек

- **Язык**: Go 1.22
- **HTTP-фреймворк**: Fiber v2 (на базе fasthttp)
- **База данных**: SQLite (pure Go реализация)
- **Конфигурация**: envconfig (через переменные окружения)
- **Логирование**: zerolog (структурированный JSON)

## Структура проекта

- `cmd/server/`: Точка входа в приложение.
- `internal/api/`: Маршрутизация, хэндлеры и middleware.
- `internal/cache/`: Реализация кэша в памяти и поискового индекса.
- `internal/storage/`: Работа с SQLite (репозитории справочников и расписаний).
- `internal/sync/`: Логика фоновой синхронизации данных.
- `internal/upstream/`: HTTP-клиент для взаимодействия с оригинальным сервером ОмГУ.
- `internal/models/`: Структуры данных API и базы данных.

## Запуск

1. Убедитесь, что настроен `.env` файл в корне проекта.
2. Установите зависимости:
   ```bash
   go mod download
   ```
3. Запустите сервер:
   ```bash
   go run cmd/server/main.go
   ```

Подробная информация о разработке доступна в `docs/core/DEVELOPMENT.md`.
