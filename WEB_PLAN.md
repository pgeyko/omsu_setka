# [Web Frontend] Зеркало расписания ОмГУ

План реализации клиентской веб-части для сервиса расписания.

## Проектирование и Архитектура

### Технологический стек
- **Фреймворк:** React (с Vite для невероятно быстрой локальной сборки и HMR) + TypeScript.
- **Стилизация:** Vanilla CSS (CSS Modules или просто грамотно структурированный CSS с использованием CSS Variables, Nesting, Grid/Flex). Отказываемся от Tailwind для создания полностью кастомного, гибкого и премиального "Wow" эффекта.
- **Стейт-менеджмент:** Zustand для простых глобальных состояний (например, "Избранное", тема "Светлая/Темная").
- **Запросы к API:** SWR или TanStack Query (React Query). Идеально подходят для кэширования, авто-обновления в фоне и индикации загрузки при работе с нашим BFF.
- **PWA (Progressive Web App):** Используем `vite-plugin-pwa`, чтобы пользователи могли установить расписание как нативное приложение на смартфон (иконка на домашний экран, работа в офлайне из кэша браузера).
- **Роутинг:** React Router v6.
- **Иконки:** Lucide React (стильные, легкие, настраиваемые).

### Дизайн система & UX
1. **Мобайл-фёрст (Mobile-First):**
   - 90% трафика — смартфоны студентов. Навигация должна быть удобной под палец (нижняя Bottom Bar навигация вместо верхнего меню).
   - Поддержка свайпов для перелистывания дней в расписании (понедельник -> вторник -> среда).
2. **Визуальная эстетика (Premium Feel):**
   - **Цветовая палитра:** Глубокая темная тема (Deep Dark Mode, например `#0f172a` фон, слегка подсвеченные неоном карточки) и чистая светлая тема. Цвет переключается системно + вручную.
   - **Глассморфизм (Glassmorphism):** Полупрозрачные навигационные панели (`backdrop-filter: blur`) и карточки для создания "дорогого", объемного интерфейса.
   - **Шрифты:** Современная типографика (Inter или Outfit), крупные и чистые заголовки, четкое разделение названий предметов, преподавателей и аудиторий.
   - **Микроанимации:** Плавное появление страниц (Fade-in, Slide-up), анимированное нажатие кнопок, анимированные скелетоны (skeletons) при загрузке данных вместо раздражающих спиннеров.
3. **Функции "Убийцы":**
   - **"Что сейчас?":** Автоматическая подсветка текущей пары, расчет и показ активного прогресс-бара времени до начала/конца пары.
   - **Мгновенный поиск:** Практически моментальный автокомплит (Autocomplete), благодаря быстрому поиску по префиксу на BFF, без лагов.
   - **Офлайн-режим:** Если пропала сеть, просмотр кэшированного расписания будет беспроблемно открываться, показывая предупреждение "вы в оффлайн-режиме, данные от <вчера>".

### Структура проекта (папка `frontend/src/`)
- `components/`
  - `ui/` — базовые переиспользуемые элементы (Button, Input, Card, Skeleton), построенные исключительно на Vanilla CSS.
  - `search/` — строка поиска с выпадающим списком.
  - `schedule/` — лента расписания, свайп-контейнер дней, карточка одной пары.
- `pages/`
  - `Home` — Главная страница (Крупная строка поиска в центре + сетка "Избранное" / "Последние запрошенные").
  - `ScheduleView` — Универсальная страница вывода расписания для Группы, Преподавателя или Аудитории.
- `styles/`
  - `global.css` — сброс стилей, базовые настройки документа.
  - `variables.css` — содержит CSS-переменные (токены) для светлой и темной тем, отступов, радиусов и теней.
- `store/` — Zustand сторы (`useFavoritesStore.ts` и `useUIStore.ts`).
- `api/` — конфигурация fetch/axios, типизированные вызовы к нашему локальному/продовому BFF API.
