# Директория разработки (dev)

В этой директории собраны все внутренние документы, планы реализации и технические спецификации проекта `omsu-mirror`.

## Состав файлов

- 📋 **[TASKS.md](./TASKS.md)**: Основной чеклист реализации. Используется для отслеживания прогресса по этапам.
- 📐 **[SPEC.md](./SPEC.md)**: Техническое задание и описание архитектуры системы (L1/L2/L3 кэш, БД, API).
- 🧠 **[GEMINI.md](./GEMINI.md)**: Контекстный файл для ИИ-ассистентов, содержащий актуальное состояние проекта и правила разработки.
- 🏗 **[BACKEND_PLAN.md](./BACKEND_PLAN.md)**: Детальный план развития серверной части (мониторинг, защита, оптимизации).
- 🎨 **[FRONTEND_PLAN.md](./FRONTEND_PLAN.md)**: План реализации и доработки веб-интерфейса.
- 🔌 **[API_DATA.md](./API_DATA.md)**: Описание структуры данных и эндпоинтов оригинального API ОмГУ.
- 🧪 **[SAMPLE_RESPONSE.md](./SAMPLE_RESPONSE.md)**: Пример ответа от upstream-сервера для отладки парсеров.

## Как использовать

1. При начале новой фичи сверяйтесь с `SPEC.md`.
2. После реализации этапа отмечайте его в `TASKS.md`.
3. При обнаружении архитектурных долгов вносите их в `BACKEND_PLAN.md` или `FRONTEND_PLAN.md`.
