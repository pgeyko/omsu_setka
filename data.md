# API расписания ОмГУ — Справочник эндпоинтов

**Base URL:** `https://eservice.omsu.ru/schedule/backend`

---

## Общая обёртка ответа

Все эндпоинты возвращают JSON в единой обёртке:

```json
{
  "success": true,
  "message": "Операция выполнена успешно",
  "data": [ /* полезная нагрузка */ ],
  "code": "SUCCESS"
}
```

---

## 1. Справочники (словари) — редко меняющиеся данные

### 1.1 Группы
- **URL:** `GET /dict/groups`
- **Описание:** Список всех учебных групп.

```json
{
  "id": 472,
  "name": "ЮСС-003-О-02",
  "real_group_id": 12246     // может отсутствовать (например, "Резерв")
}
```

**Поля:**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Внутренний ID записи в справочнике |
| `name` | string | Полное название группы |
| `real_group_id` | int? | Внешний ID для запроса расписания (nullable) |

**Особенности:**
- По коду группы можно определить факультет, курс и форму обучения.
- Записи-заглушки типа `"Резерв"` не имеют `real_group_id`.

---

### 1.2 Аудитории
- **URL:** `GET /dict/auditories`
- **Описание:** Справочник аудиторий и корпусов.

```json
{
  "id": 3024,
  "name": "1-1",
  "building": "1"
}
```

**Поля:**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Уникальный ID помещения |
| `name` | string | Название аудитории (формат: корпус-номер) |
| `building` | string | Номер/код корпуса |

**Типы локаций:**
- Стандартные: `"4-308"`, building `"4"`
- Специализированные: `"(6-115) Бассейн"`, `"Тренажерный зал"`
- Виртуальные: `"Дистант"`, `"онлайн"`, `"в записи"` — building = `"0"`
- Корпуса: цифровые (`1`, `2`, `4`, `5`, `6`, `7`, `11`) и буквенные (`Г`)
- Спецсимволы: `"1-102*"` — временная/измененная локация

---

### 1.3 Преподаватели
- **URL:** `GET /dict/tutors`
- **Описание:** Справочник преподавателей.

```json
{
  "id": 303,
  "name": "Озол Виктория Александровна"
}
```

**Поля:**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Уникальный ID преподавателя |
| `name` | string | ФИО |

**Особенности:**
- Служебные записи: `"- "`, `"_"`, `"онлайн в записи"`
- Возможны дубликаты с разным написанием ФИО

## 2. Расписание — динамические данные

### 2.1 Расписание группы
- **URL:** `GET /schedule/group/{real_group_id}`
- **Пример:** `/schedule/group/15103`

### 2.2 Расписание аудитории
- **URL:** `GET /schedule/auditory/{id}`
- **Пример:** `/schedule/auditory/3024`

### 2.3 Расписание преподавателя
- **URL:** `GET /schedule/tutor/{id}`
- **Пример:** `/schedule/tutor/303`

### Структура ответа расписания

```json
{
  "day": "04.05.2026",
  "lessons": [
    {
      "id": 54578562,
      "time": 1,
      "faculty": "ФМС",
      "lesson": "Алгебра Лек",
      "type_work": "Лек",
      "teacher": "Бесценный Игорь Павлович",
      "group": "МБС-501-О-01",
      "auditCorps": "4-308",
      "subgroupName": "МБС-501-О-01/1",
      "publishDate": "06.04.2026 13:26:31"
    }
  ]
}
```

**Поля занятия:**
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Уникальный ID занятия |
| `time` | int | Номер пары (1-7) |
| `faculty` | string | Код факультета |
| `lesson` | string | Название дисциплины + тип |
| `type_work` | string | Тип занятия: `Лек`, `Прак`, `Лаб` |
| `teacher` | string | ФИО преподавателя |
| `group` | string | Название группы |
| `auditCorps` | string | Аудитория-корпус |
| `subgroupName` | string? | Подгруппа (опционально, для лаб) |
| `publishDate` | string | Дата публикации записи |

---

## 3. Связи данных

```
dict/groups.id → schedule/group/{real_group_id}
dict/auditories.id → schedule/auditory/{id}
dict/tutors.id → schedule/tutor/{id}
```

### Частота обновлений
- **Справочники** (groups, auditories, tutors): меняются редко (начало семестра)
- **Расписание**: может меняться ежедневно (замены, переносы)
